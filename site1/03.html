כח הצבים - חלק 3

שלב 10: למה דוקא ריבוע?

לא סיימנו עדיין, למה שלא נשנה את הפונקציה ככה שתוכל לצייר כל צורה!

בעצם כל הפעולות של turtle שהשתמשנו בהן הן פשוט פונקציות, וכמו עם הפעולה forward שמקבלת מספר כדי לדעת כמה לזוז, גם אנחנו יכולים לגרום לפונקציות שלנו לקבל מספרים.

  1. בואו נשנה את הקוד מהתרגיל הקודם בשביל שייראה כך:

    from turtle import *
shape("turtle")

def poly(sides, length):
    angle = 360/sides

    for n in range(sides):
            forward(length)
            right(angle)

pencolor('red')
poly(4, 100)
right(180)
pencolor('blue')
poly(3, 150)

done()

  2. תריצו את הקוד ותנסו להבין מה קורה

    בואו ניקח את זה לאט, כי זה באמת די מגניב. במקום שנגדיר את המשתנים בתוך הפונקציה, אמרנו לפונקציה שהיא תקבל מספרים מבחוץ בכל פעם שהיא רצה ותשים אותם בעצמה בתוך המשתנים sides ו-length.

    בעצם, לקחנו את ההגדרות של הצורה מתוך הפונקציה, החוצה לאיפה שמחליטים אשכרה לצייר את הצורה. ומה שיצא זה שאנחנו יכולים עכשיו באמצעות שורה אחת להחליט לצייר כל צורה שנרצה. די מגניב, לא? ככה אנחנו יכולים ללמד את המחשב טריקים מורכבים חדשים ואז לבקש ממנו לעשות אותם מתי שבא לנו ואיך שבא לנו.

היכולת הזאת להגדיר פונקציות שיכולות לעשות דברים קצת שונים בכל פעם בהתאם למה שאנחנו רוצים היא מרכיב מאוד מרכזי ואחד הדברים הכי חשובים בתכנות.

שלב 11: צורות מקווקוות

  1. בואו עכשיו נחזור לעניין הקווים המקווקוים שהתחלנו בשלב 8. תכתבו את הקוד הבא:

    from turtle import *
shape("turtle")

length = 200
for num in range(8):
    forward(length/16)
    penup()
    forward(length/16)
    pendown()

done()

  2. זה אמור לצייר קו מקווקו נחמד. תריצו ותוודאו שהכל סבבה.

  3. עכשיו שאנחנו יודעים לעשות קווקווים, אנחנו יכולים לגרום לצב לעשות צורה שלמה מקווקוות במקום רק קו אחד. בואו ניקח את הקוד של poly מהשלב הקודם ונחליף את השורה של ה-foreward בקוד שעושה קווקווים.

  4. תזינו את הקוד הבא:

    from turtle import *
shape("turtle")

def dashpoly(sides, length):
    angle = 360/sides

    for n in range(sides):
        for num in range(8):
            forward(length/16)
            penup()
            forward(length/16)
            pendown()

        right(angle)

pencolor('red')
dashpoly(4, 100)
right(180)
pencolor('blue')
dashpoly(3, 150)

done()

  5. תריצו את הקוד ותראו מה הוא עושה.

    כמו שאמרנו קודם, יש לנו שתי לולאות, אחת בתוך השניה. הלולאה החיצונית for n in range(sides) אחראית לכך שנעשה את כל הצלעות של הצורה ועבור כל אחת מהצלעות היא מריצה את הלולאה הפנימית for num in range(8) שאחראית על הקווקווים.


    בשיעור שעבר אמרתי שעוד נדבר על איך עובדות לולאות, אז הנה, בואו נדבר קצת עכשיו ואני מבטיח שבשיעור הבא אסביר אפילו יותר. הטריק הוא כזה, כשאנחנו עושים לולאת for, פייתון צריך להשתמש במשתנה כלשהו בשביל לדעת באיזה צעד של הלולאה הוא נמצא עכשיו וזה התפקיד של המילה שכתובה אחרי for, זה השם של המשתנה שאנחנו בוחרים (ואפשר לבחור כל דבר שרוצים).

    בקוד שלנו, בלולאה החיצונית השתמשנו במשתנה n כדי לספור את הצלעות ובלולאה הפנימית השתמשנו במשתנה num כדי לספור את הקווקווים. כשעושים לולאה בתוך לולאה, חייבים לבחור בשמות משתנים שונים, אחרת פייתון יתבלבל (וכנראה גם אנחנו).

  6. אני באמת מבטיח שנדבר עוד על לולאות בשיעור הבא, אבל אם בא לכם קצת לשחק עם המשתנה של הלולאה, תנסו להריץ את הקוד הזה:

    for a_nice_number in range(8):
    print(a_nice_number)
  7. כשתריצו, אתם אמורים לקבל שמונה שורות של מספרים על המסך. למה דוקא המספרים האלו? כי פייתון אוהב להתחיל מאפס.

שלב 12: אותו דבר אבל פשוט יותר

  1. בואו נשתמש בפונקציות בשביל לפשט את זה עוד יותר. אנחנו ניקח את הקוד של התוכנה הקודמת ונפרק אותו לשתי פונקציות (אחת שמשתמשת בשנייה) כך שהוא יהיה יותר ברור.

    from turtle import *
shape("turtle")

def dashforward(length):
    for num in range(8):
        forward(length/16)
        penup()
        forward(length/16)
        pendown()

def dashpoly(sides, length):
    angle = 360/sides
    for n in range(sides):
        dashforward(length)
        right(angle)

pencolor('red')
dashpoly(4, 100)
right(180)
pencolor('blue')
dashpoly(3, 150)

done()

  2. תריצו את הקוד ואתם אמורים לקבל בדיוק אותו הדבר.

טיפ:

מה שעשינו עכשיו נקרא ריפקטורינג (Refactoring) - התהליך שבו משנים את הקוד כך שיהיה פשוט יותר, ברור יותר ונח יותר לעבודה מבלי לשנות את מה שהוא עושה. כשכותבים תוכניות גדולות, שווה מאוד להשקיע את הזמן כדי לנסות להבין אם יש דברים שאפשר לפשט ובפרט אם אפשר להוציא רכיבים לפונקציות מסודרות. אם לא עושים את זה, אז בשלב מסויים, הקוד הופך להיות כל-כך מורכב שכמעט ואי-אפשר לעבוד עליו.

שלב 13: קצת אקראיות

עכשיו שלב בונוס לקראת סיום השיעור, בואו ננסה להוסיף קצת אקראיות בשביל שכל פעם שנריץ את התוכנה שלנו יקרה משהו אחר ומפתיע. אנחנו יכולים לבקש מהמחשב להגריל מספר בשבילנו או לבחור בשבילנו צבע - זה קצת כמו לזרוק קובייה.

  1. תצרו קובץ חדש ותרשמו את הקוד הבא:

    from turtle import *
from random import randrange, choice
shape("turtle")

colors = ['red', 'blue', 'green']

def poly(sides, length):
    angle = 360/sides
    for n in range(sides):
        forward(length)
        right(angle)


for count in range(10):
    pencolor(choice(colors))
    right(randrange(0, 360))
    poly(randrange(3, 9), randrange(20, 50))

done()

  2. תשמרו ותריצו - התוכנה הזאת אמורה לצייר בעצמה עשר צורות שונות בצבעים שונים ובגדלים שונים.

    השורה from random import randrange, choice מאפשרת לנו להשתמש בשתי פונקציות חדשות מספרייה אחרת שנקראת random (עד עכשיו השתמשנו רק בספרייה turtle).
    הפונקציה randrange גורמת לפייתון לבחור מספר אקראי בין שני מספרים שאנחנו בוחרים. למשל randrange(1, 10) יגרום לפייתון לבחור מספר שלם בין 1 ל-9 (פייתון יתחיל ב-1 ויפסיק לפני שיגיע ל-10).


    הפונקציה choice גורמת לפייתון לבחור מספר מתוך רשימה. רשימה בפייתון (list) זה אוסף של איברים שבאים אחד אחרי השני בתוך סוגריים מרובעים, למשל [1, 2, 3], או רשימת הצבעים בקוד שלנו - ['red', 'blue', 'green']. גם על רשימות אני מבטיח לדבר עוד בשיעור הבא.


    בזכות השימוש שלנו בפונקציות choice ו-randrange, אנחנו גורמים למחשב לבחור בשבילנו צבעים, צורות וגדלים שונים בכל פעם שנריץ את התוכנה.

תנסו אולי להוסיף עוד צבעים ולשחק עם המספרים. מה תוכלו עוד לעשות בשביל שזה יהיה אפילו יותר מגניב?

קבלו טיפ: אם אתם מציירים הרבה דברים שונים, יהיה לכם אולי קצת יותר נח להשתמש בפונקציה exitonclick במקום ב done, כי אז אפשר לסגור את הציור עם לחיצה בכל מקום ולא רק על האיקס.

שיעורי בית - השמים הם הגבול

למדתם בינתיים לזוז קדימה ואחורה, שמאלה וימינה; לעשות לולאות, לשנות צבעים, למלא צורות ועוד ועוד. מה תעשו עם כל הכוחות האלו?

האם תוכלו לצייר בית גדול עם הרבה חדרים? ציפור עם כנפיים צבעוניות? נחש מפחיד? חתול? כלב? אולי אריה? אולי הכל ביחד? אולי בכלל משהו אחר שאתם תמציאו?

תנסו להשתמש בכלים ובטריקים שלמדתם כדי לבנות ציור יפה שמורכב מהרבה צורות ותשתדלו שגם הקוד שאתם כותבים יהיה יפה.

נ.ב.
אם במקרה בא לכם להשתמש במשהו שעדיין לא למדנו, זה גם סבבה. הנה שוב הקישור לכל הדברים שאפשר לעשות עם הצב (אבל זה רק אם רוצים, ולצערי יש את זה רק באנגלית).
