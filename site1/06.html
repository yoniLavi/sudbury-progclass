6
ניהול הקוד עם גיט

שלב 27: מבוא לניהול גרסאות

עכשיו שאנחנו מתחילים לעבוד על תוכנות יותר גדולות, שווה שנלמד על כלי שיעזור לנו לארגן ולשתף את העבודה שלנו - גיט. גיט הוא כלי לניהול קבצי קוד שעוזר לתעד מה המשמעות של כל שינוי, לשתף קוד עם אחרים, לנסות שינויים בקוד מבלי לפגוע במה שכבר עשינו וכו...
  1. גיט זה כלי לניהול גרסאות והמון המון אנשים משתמשים בו כדי לשתף את הקוד שהם כותבים עם העולם. אתר הגיט הגדול בעולם נקרא גיטהאב (github) ובו יש תוכנות שמיליוני אנשים העלו בשביל לשתף אותן לכל העולם. כל העניין זה נובע מגישה שנקראת "קוד פתוח" (Open Source) שאומרת שאם נראה לעולם את הקוד שלנו, זה יעזור לאחרים ללמוד, וזה יאפשר להם לעזור לנו להשתפר וללמוד וכך כולם ירוויחו. ומבחינה יותר פרקטית, זה מאפשר לכולם למצוא ולפתור באגים יותר בקלות בגלל שהכל פתוח.
  2. תעשו עכשיו סיבוב של כמה דקות בgithub ותסתכלו קצת איזה סוגים של דברים יש שם ומה האתר מאפשר לעשות. בשלב הזה אתם עדיין לא צריכים להירשם, הכל נגיש בצורה חופשית. שימו לב שהאתר מחולק לפי יוזרים ולכל יוזר יכולים להיות הרבה פרוייקטים שכוללים את כל הקבצים והתיקיות שנדרשים בשביל להפעיל תוכנה כלשהי. פרוייקט נקרא בגיט Repository (מאגר).
  3. אני קצת שיקרתי קודם. גיט וגיטהאב זה לא רק בשביל תוכנות. אפשר להשתמש בגיט בשביל כל דבר שאפשר להציג כטקסט ובין השאר:
    • ספרים
    • החוקה של ארה"ב
    • נתונים על שילוב דמויות נשיות בסרטים הוליוודיים (לפי מבחן בקדל)
    • ואפילו רצף DNA של אנשים (זהירות, מדובר בקובץ עצום בגודלו).

שלב 28: נתחיל לנהל את הקבצים שלנו

  1. עכשיו בואו נעזוב קצת את האינטרנט וננסה להבין איך גיט עובד על המחשב שלנו. תפתחו את bash ותריצו את שתי הפקודות הבאות: 
    mkdir cats_and_stuff
cd cats_and_stuff
    גיט יודע רק לנהל פרוייקטים לפי תיקיות, אז יצרנו תיקיה חדשה בשם cats בעזרת הפקודה mkdir ואז נכנסנו לתוכה. כל מה שנעשה מעכשיו יהיה בתוך התיקיה הזאת.
  2. השלב הבא הוא לבקש מגיט להתחיל לנהל את התיקיה הזאת בעזרת הפקודה הבאה: 
    git init
  3. זהו, עכשיו התיקיה הפכה להיות repository של גיט ואנחנו יכולים להשתמש בפקודות של גיט בשביל לנהל את הקבצים שלנו שבתוכה.
  4. והפקודה הראשונה (ואולי הכי חשובה) שנכיר היא זו: 
    git status
    פקודה זו מספרת לנו מה קורה כרגע ובמקרה הזה היא רק אומרת שאין שם שום דבר מעניין. לא נורא, עוד מעט נתחיל להכניס דברים מעניינים.

    הערה: כשתבואו בעתיד להתחיל להשתמש בגיט על תיקיה שכבר יש בה קבצים, כל השלבים נשארים אותו דבר.

שלב 29: מתחילים להוסיף קבצים

  1. בואו ניצור קובץ קטן בתיקיה שלנו: 
    echo "hello octocat" >octocat.txt
    זוכרים את הפקודה echo שכותבת טקסט למסך? אז מה שעשינו עכשיו זה שהשתמשנו בחץ בשביל להפנות את הטקסט לקובץ בשם octocat.txt, ובגלל שהקובץ לא היה קיים עדיין, bash יצר אותו על המקום. מגניב, לא?
  2. בואו נבדוק מה השתנה בעזרת: 
    git status
    גיט אמור להגיד לכם עכשיו שיש קובץ חדש שהוא מצא.
  3. אבל עד שלא נגיד לגיט לנהל את הקובץ הזה הוא יתעלם ממנו. תריצו את הפקודה הבאה בשביל להגיד לגיט שהקובץ מעניין: 
    git add octocat.txt
    הפקודה git add הוסיפה את הקובץ לרשימת הדברים שמעניינים את גיט, אבל עדיין לא עשתה שום דבר חוץ מזה. כרגיל, שווה להריץ בדיקת סטטוס כדי לראות מה השתנה: 
    git status
  4. אוקי, אז אמרנו לגיט שיש קובץ מעניין, אבל לא אמרנו לו למה זה מעניין. אז בואו נריץ את הפקודה הבאה: 
    git commit -m "Added an octocat story"
    מה שאמרנו לגיט עכשיו הוא בעצם - קח את הקובץ הזה שאתה זוכר שהוא מעניין ותזכור את השינויים שעשינו לו בתור זה שהוספנו סיפור על האוקטוקט (למי מכם שלא מכיר, מדובר ביצור חביב עם גוף של חתול ורגליים של תמנון שהוא הסמל הרשמי של גיטהאב). ככה כל פעם שאנחנו מסיימים לעשות שינויים משמעותיים בקבצים שלנו, נספר לגיט מה עשינו והוא ישמור את זה בשבילנו.
  5. וגם הפעם נריץ בדיקת סטטוס: 
    git status
    מה שאולי יפתיע אתכם זה שהסטטוס אומר שאין בכלל שום דבר לספר. חזרנו בעצם לנקודת ההתחלה שבה גיט יודע הכל על התיקיה. בואו נחזור על מה שעשינו בשלב הזה כי זה הבסיס לכל העבודה עם גיט: עשינו שינויים בקובץ או כמה קבצים בתיקיה שלנו, אמרנו לגיט איזה קבצים מעניינים (בכך הוספנו אותם לאזור שנקרא staging area) ואז לבסוף יצרנו קומיט שמתאר את השינויים בקבצים.

שלב 30: עוד עבודה על הקבצים

  1. בואו נמשיך לעבוד על פרוייקט החתולים שלנו. נוסיף עוד שני קבצים וגם נשנה את הקובץ הישן: 
    echo "hello blue" >blue_cat.txt
echo "hello red" >red_cat.txt
echo "hello awesome octocat" >octocat.txt
  2. נריץ שוב סטטוס כדי לראות מה השתנה מאז הקומיט האחרון: 
    git status
    ואם אנחנו רוצים לראות יותר בתכל'ס מה ההבדלים שקרו, נוכל גם להריץ את הבדיקה הזו: 
    git diff
    הכל נראה סבבה, נכון? אם לא, אתם יכולים לעשות כל שינוי נוסף שבא לכם בקבצים האלו.
  3. אוקי, אז בואו נגיד לגיט שכל הקבצים שמסתיימים בסיומת txt הם מעניינים וניצור קומיט חדש: 
    git add *.txt
git commit -m "Fixed missing cat stuff"
    שימו לב שהכוכבית הזאת שהשתמשנו בה אמרה ל-bash לבחור את כל הקבצים עם הסיומת הזאת, כשלא משנה איך הם מתחילים.
  4. הפרוייקט שלנו כבר הופך להיות די רציני. בואו נראה מה הם כל הדברים שעשינו בינתיים בעזרת: 
    git log
    אתם אמורים לראות עכשיו סיכום על שני הקומיטים שעשינו.

שלב 31: לשתף עם כולם בגיטהאב

  1. עכשיו שיש לנו פרוייקט מטורף שאנחנו מרוצים ממנו, בואו נשתף אותו עם העולם. תיכנסו שוב לגיטהאב ותיצרו לעצמכם חשבון (בחינם) בעמוד הזה. שימו לב שאפשר לשלם לגיטהאב כדי שישמרו על הדברים שלכם בסודיות, אבל אין הרבה סיבות לעשות את זה, אלא אם אתם מתכוונים למכור את התוכנה שלכם ואתם מפחדים מאוד שיגנבו לכם רעיונות.
  2. השלב הבא, אחרי שיצרתם את החשבון הוא ליצור פרוייקט (repository) חדש. זה לא משנה איך תקראו לו (זה לא צריך להיות כמו התיקיה), אבל אני אניח שקראתם לו weird_cats. גם שאר האפשרויות שם לא משנות לנו כרגע, אז תבחרו את מה שבא לכם.
  3. כשהפרוייקט החדש יווצר, תועברו לעמוד של הפרוייקט ותוכלו לראות בצד ימין תיבת טקסט חמודה שנקראת HTTPS clone URL ובה הכתובת של הפרוייקט הזה בגיטהאב.

    תעתיקו לעצמכם את הכתובת הזו ותכניסו אותב במקום הכתובת בפקודה הבאה ואז תריצו את זה בתוך התיקיה שלכם: 
    git remote add origin https://github.com/sudbury_cat_stuff/weird_cats.git
git push -u origin master
    בכך אמרנו לפרוייקט הגיט בתיקיה שלנו שיש לו עותק חדש באינטרנט, שאנחנו מבקשים לקרוא לו origin ושאנחנו רוצים לדחוף לעותק הזה בגיטהאב את כל הדברים שעשינו עד כה.

    שימו לה שבשלב הזה גיטהאב יבקש מכם שם משתמש וסיסמא - זה לא כאילו שכל אחד יכול לשנות את הפרוייקט שלכם מבלי לשאול אתכם.

    הערה: מה שכן יכול כל אחד, זה לעשות עותק משלו לפרוייקט של מישהו אחר. על כל פרוייקט שאתם רואים בגיטהאב מופיעה אפשרות fork למעלה מימין. אם תלחצו על זה, יווצר בחשבון שלכם בגיטהאב עותק שלם של הפרוייקט הזה שתוכלו לעשות איתו מה שבא לכם. ואם תעשו דברים מגניבים עם הפרוייקט, אולי תוכלו לשכנע את המפתח המקורי שהוא יעתיק את השינויים שלכם לעצמו.
  4. זהו, בשלב הזה הפרוייקט בגיטהאב מקושר לפרוייקט אצלכם במחשב ועל כל שינוי אצלכם במחשב אתם יכולים לעדכן את גיטהאב פשוט ע"י: 
    git push
    ואם במקרה הפרוייקט בגיטהאב השתנה (נניח ממחשב אחר שעבדתם עליו) ואתם רוצים לעדכן את המחשב הזה, אתם יכולים להריץ: 
    git pull
    אתם יכולים להריץ את שתי הפקודות האלו כמה שבא לכם ואם אין לגיט שום דבר חדש לעשות, אז הוא פשוט לא יעשה כלום.

שלב 32: מה עכשיו?

  1. בזאת כמעט סיימנו את המבוא לגיט. לגיט יש המון אפשרויות מתקדמות יותר ואפשר לעשות איתו בערך כל דבר. בפרט, אם בא לכם לדעת יותר, למשל מה זה branch, או merge או pull request, תשאלו ואשמח לספר לכם את כל מה שאני יודע. אבל מה שלמדנו היום אמור די להספיק לנו בשביל להתחיל לעבוד עם גיט. ואת מה שחסר נוכל להשלים בהתאם לצורך.
  2. אה, הנה משהו שאפשר להשלים כבר עכשיו. תראו לחברים שלכם את הפרוייקט שלכם ותציעו להם למשוך אותו למחשב שלהם. בשביל לחסוך זמן, במקום ליצור תיקיה, לעשות git init וכו... אפשר להעתיק פרוייקט מגיטהאב בפקודה אחת (תחליפו את הכתובת של הפרוייקט בכתובת שלכם): 
    git clone https://github.com/sudbury_cat_stuff/weird_cats.git
  3. בשלב הזה, בשביל ללמוד לעבוד עם גיט, כדאי שתנסו להשתמש בו כדי לנהל את הקוד של המשחק שלכם מהשיעור שעבר. (וכל פרוייקט אחר שתרצו לעבוד עליו)
