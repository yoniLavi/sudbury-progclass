היכרות ראשונה עם הווב

שלב 33: מבוא - מהו הווב?

כמו שסיפרתי בשיעור הקודם, הווב הוא דרך להעביר מסמכים (בעיקר עמודי html, באמצעות פרוטוקול http) דרך רשת האינטרנט. בווב יש שני סוגי משתתפים - שרת ודפדפן.
  1. דפדפן (נקרא גם לקוח) - תוכנה שאנחנו הגולשים משתמשים בה כדי לבקש מסמכים.
  2. שרת (נקרא גם אפליקציית ווב) - תוכנת מחשב (שלרוב יושבת במחשב אחר שגם הוא נקרא שרת) שאחראיות לספק ללקוחות את דפי הווב שהם מבקשים.
  3. הדרך שבה דפדפן מבקש מסמך משרת היא ע"י URL (Uniform Resource Locator). ה-URL היא כתובת שאומרת איזה מסמך הדפדפן רוצה לקבל והיא בנויה משני חלקים עיקריים: כתובת השרת, למשל - en.wikipedia.org וכתובת המסמך, למשל - /wiki/Sudbury_school
    אז אם נבקש בעזרת הדפדפן את ה-URL הזה http://en.wikipedia.org/wiki/Sudbury_school, נקבל מהשרת של וויקיפדיה את הדף על בתי הספר של מודל סאדברי.

שלב 34: פונקציות שמחזירות דברים

  1. עד עכשיו, יצרנו רק פונקציות שעושות פעולות, כמו להדפיס טקסט על המסך או לצייר מרובעים. אבל בעולם האמיתי, רוב הפונקציות המעניינות לא מתקשרות עם העולם החיצון, אלא רק מחשבות דברים בשביל פונקציות אחרות.
  2. פונקציות כאלו משתמשות בפקודה return. הפקודה הזאת מפסיקה את ריצת הפונקציה ומחזירה את הערך ששמים בה. למשל תנסו להריץ את זה: 
    def doubler(num):
    return num * 2
    print("No one will ever read this")

x = 21
double_x = doubler(x)
print(double_x)
    כשמריצים את הפקודה double_x = doubler(x), הפונקצייה doubler מחשבת את הערך של x כפול 2, ואז הוא נכנס למשתנה double_x.
  3. שימו לב שבעבר נעזרנו כבר בפונקציות שמחזירות ערכים בעזרת return וביניהן: input, choice ו-randrange.
  4. הנה שימוש לא מוצלח ב-return. תריצו את התוכנה הבאה, תבינו למה היא לא עובדת ותנסו לתקן אותה: 
    def foo(num):
    if num != 42:
        return "This is not the answer"
        return "This is the answer to the question of life, the universe and everything!"

print(foo(42))

שלב 35: נתקין את פלאסק

בשלב הבא נתחיל לבנות שרת ווב, אבל לפני זה צריך להכין את הכלי שאיתו נעבוד.
  1. אנחנו לא נבנה את השרת מאפס אלא נשתמש בספרייה שנקראת פלאסק (flask). פלאסק היא ספרייה מסוג שנקרא פריימוורק (Web Framework). המטרה של פריימורק היא לספק למתכנת את כל הכלים שקשורים לתקשורת עם האינטרנט והדפדפן, כך שלמתכנת נשאר רק לבנות את התוכן שיוחזר עבור כל URL.
  2. פייתון מגיע כבר "מהקופסא" עם הרבה ספריות כמו turtle ו-random, אבל יש המון ספריות מעולות נוספות שאפשר להוריד מהאינטרנט. הדרך הכי נוחה בפייתון להתקין ספריות היא כלי קטן שנקרא פיפ (pip). תנסו לכתוב ב-bash pip כדי לראות אם אולי פיפ כבר מותקן אצלכם, ואם כן, אתם יכולים לדלג לסעיף הבא. אם אין לכם את פיפ, תורידו את הקובץ הבא get-pip ואז תתקינו אותו בעזרת הפקודה הבאה ב-bash (בתיקיה שאליה הורדתם את הקובץ): 
    python get-pip.py
  3. ועכשיו שיש לכם את פיפ, אתם יכולים להתקין את פלאסק בעזרת הפקודה הבאה ב-bash 
    pip install flask

שלב 36: תוכנה ראשונה בפלאסק

עכשיו אנחנו מוכנים להתחיל לבנות שרת ווב פשוט וכך להכיר את פלאסק.
  1. שרתי ווב הם דומים קצת לתוכנות שורת הפקודה שהכנו בשיעור 4, אבל הבדל עיקרי הוא שאנחנו לא צריכים לעשות לולאה כמו שעשינו אז, אלא פלאסק דואג בשבילנו לכל זה, ואנחנו רק צריכים לכתוב את הפעולות אשר יקרו עבור כל URL.
  2. ככה נראה שרת פשוט בפלאסק שאומר שלום לעולם. תיצרו תיקיה חדשה ותשמרו את הקוד הזה בקובץ בתוכה שתקראו לו helloflask.py (אנחנו נשתמש בשם הזה אח"כ): 
    import flask

app = flask.Flask(__name__)

@app.route('/')
def hello_world():
    return 'Hello World!'

app.run(debug=True)
    תריצו את הקובץ הזה ב-bash ותיכנסו בדפדפן ל- http://127.0.0.1:5000/ (הכתובת 127.0.0.1 היא תמיד הכתובת של המחשב המקומי שלכם ו-5000 זה הפורט שפלאסק מעדיף). שימו לב שפלאסק יריץ את השרת בלולאה אינסופית עד שתהרגו את התוכנה בעזרת Ctrl+C.
  3. בואו ננסה להבין מה מכיל הקוד שכתבנו. השורה app = flask.Flask(__name__) מגדירה אפליקציית פלאסק חדשה, כאשר המשתנה המיוחד __name__ מכיל את שם הקובץ שאנחנו בתוכו. עכשיו יש לנו במשתנה app את כל מה שאנחנו צריכים בשביל לעבוד על פלאסק.
    דבר אחר שלא ראינו כמותו עד כה הוא השורה שמתחילה בשטרודל - @app.route('/'). השורה הזאת נקראת דקורטור (כי היא מקשטת פונקציה) והיא דרך שבה אנחנו אומרים לפלאסק מה לעשות עם הפונקציה שמתחת. במקרה הזה, אנחנו אומרים לפלאסק שהפונקציה הזאת אחראית לשרת את הכתובת / (שורש האתר).
    השורה האחרונה app.run(debug=True) אחראית פשוט להריץ את השרת.

שלב 37: בואו נכין עוד עמודים

עכשיו שהבנו את העיקרון, בואו נראה מה עוד אפשר לעשות.
  1. אז בואו נוסיף עוד עמוד. תשנו את הקוד כך שיכיל את הפונקציה החדשה ותשמרו: 
    import flask

app = flask.Flask(__name__)

@app.route('/')
def hello_world():
    return 'Hello World!'

@app.route('/hello')
def hello():
    return "Hello Yoni"

app.run(debug=True)
    שימו לב שהפעם שמנו בדוקרטור את הכתובת /hello, אז עכשיו אתם יכולים לגלוש בדפדפן לכתובת http://127.0.0.1:5000/hello ותראו את העמוד החדש.
    אגב, שימו לב שלא היה צורך לעצור את התוכנה ולהריץ אותה מחדש. הסיבה לכך היא שכתבנו בשורה האחרונה debug=True וכך קיבלנו מפלאסק כל מיני אפשרויות שעוזרות לנו בפיתוח.
    חוץ מזה, אם עדיין לא שיניתם את השם בקוד בשביל שיגיד שלום לכם ולא לי, למה אתם מחכים?
  2. רגע, למה בעצם שהשרת שלנו יגיד שלום רק למישהו אחד? בואו נעשה שהוא יוכל להגיד שלום לכל אחד. בשביל זה, אנחנו יכולים להגיד לדקורטורים של פלאסק שיקבלו פרמטרים, אז בואו נשנה את הפונקציה והדקורטור: 
    import flask

app = flask.Flask(__name__)

@app.route('/')
def hello_world():
    return 'Hello World!'

@app.route('/hello/<name>')
def hello(name):
    return "Hello there %s" % name

app.run(debug=True)
    שינינו את הדקורטור כך שיקבל את כל הכתובות מהצורה /hello/<name>, כלומר שיכול להופיע אחרי hello כל שם. אז בואו תנסו לגלוש ל-http://127.0.0.1:5000/hello/Yoni. מה שקרה פה זה שהפרמטר name נכנס מה-URL לפונקציה שלנו ואז להודעה שמוחזרת לדפדפן. תנסו להגיד שלום לעוד שמות.
    שימו גם לב, שב-URLים יש הבדל בין אותיות קטנות לגדולות, אז השרת יודע להבדיל למשל בין http://127.0.0.1:5000/hello/Yoni לבין http://127.0.0.1:5000/hello/yoni

שלב 38: פלאסק והווב

רגע, בשיעור שעבר דיברנו על הווב, אבל בינתיים זה סתם נראה כמו עוד תוכנה בפייתון. איפה נכנס ה-html?
  1. אז ככה, הדפדפן באמת מצפה שתתנו לו קוד html. אם השרת שלכם מחזיר סתם טקסט, אז הדפדפן פשוט מתייחס לזה כ-html לא מעוצב. בשיעור הבא, נלמד על דרך חכמה לארגן html בשרת שלנו, אבל בינתיים בואו פשוט נשנה את הפונקציות שלנו כדי שיחזירו קטעי html ויעצבו את השם שלנו: 
    import flask

app = flask.Flask(__name__)

@app.route('/')
def hello_world():
    return 'Hello World!'

@app.route('/hello/<name>')
def hello(name):
    return "Hello there <b>%s</b>" % name

@app.route('/bye/<name>')
def bye(name):
    return "Goodbye <i>%s</i>" % name

app.run(debug=True)
    אולי אתם יכולים לעשות עמודים יותר יפים? תנסו לשחק עם ה-html ולעשות משהו מגניב. למשל, אולי אתם יכולים להוסיף תמונות לעמוד?
  2. אוקי, אז אפשר להשתמש ב-html, אבל אולי זה נראה לכם שבינתיים לא עשינו שום דבר מעניין ולמה בכלל אנחנו צריכים את פייתון בשביל דברים כל-כך פשוטים. אם זה מה שאתם חושבים, אז, קודם כל, אתם צודקים.
    היופי בשימוש בפייתון הוא שאנחנו יכולים להכניס כל לוגיקה שאנחנו רוצים בתוך קוד הפייתון שלנו בשביל לקבל החלטות לגבי מה להחזיר לדפדפן. למשל, בואו ניקח את פונקציית המספרים הגדולים משיעור 5: 
    import flask

app = flask.Flask(__name__)

@app.route('/')
def hello_world():
    return 'Hello World!'

@app.route('/hello/<name>')
def hello(name):
    return "Hello there <b>%s</b>" % name

@app.route('/bye/<name>')
def bye(name):
    return "Goodbye <i>%s</i>" % name

@app.route('/number/')
def large_number(number_name):
  number = int(number_name)

  if number > 100000:
    return "Thank you, %s is good enough for me" % number
  else:
    return "Are you kidding me?! That's way too small"

app.run(debug=True)
    ככה אנחנו יכולים לקבל בשרת החלטות בהתאם לקלט מהמשתמש ולהחזיר לו את מה שאנחנו מחליטים שהוא אמור לראות.
    תנסו אולי לכתוב עוד פונקציה שמקבלת החלטות. למשל, תנסו לכתוב פונקציה שמחכה לסיסמא ומספרת לכם סוד רק אם אתם יודעים את התשובה הנכונה.
  3. יצא לכם עד עכשיו להקליד בדפדפן כתובת שהשרת לא מכיר ולקבל שגיאה? למשל, האם ניסיתם לבקר ב-http://127.0.0.1:5000/nosuchpage?
    אם כן, זה אומר שקיבלתם את עמוד 404 של פלאסק שסיפר לכם שאין כזאת כתובת. אבל זה לא עמוד מעניין במיוחד. בואו נשנה את עמוד השגיאה על זה למשהו יותר אינפורמטיבי: 
    import flask

app = flask.Flask(__name__)

@app.route('/')
def hello_world():
    return 'Hello World!'

@app.route('/hello/<name>')
def hello(name):
    return "Hello there <b>%s</b>" % name

@app.route('/bye/<name>')
def bye(name):
    return "Goodbye <i>%s</i>" % name

@app.errorhandler(404)
def page_not_found(error):
    return "404: You're funny, there's no such page", 404

app.run(debug=True)
    תנסו לבקר שוב ב-http://127.0.0.1:5000/nosuchpage ותראו את השגיאה החדשה.
    אם אתם שואלים את עצמכם מה הקטע עם המספר הזה, אז ככה: השרת תמיד צריך להחזיר על כל בקשה לא רק טקסט אלא גם סטטוס מספרי. הסטטוס הזה אומר לדפדפן (ולגולשים) האם הכל בסדר, או שאולי כדאי לעשות משהו מיוחד. אם לא אומרים לשרת שום דבר, אז הוא מחזיר סטטוס 200 שאומר "הצלחה", אבל יש גם הרבה סטטוסים שמספרים על שגיאות מעניינות. סטטוס 404 שראינו, למשל, מספר על כך שאין שום דבר בכתובת הזאת.
    ויש עוד המון סטטוסים אחרים. קבלו פירוט של כל הסטטוסים האפשריים בעזרת חתולים

שלב 39: heroku

בנינו שרת ווב, אבל בינתיים הוא נגיש רק מהמחשב שלנו, והרי אמרתי שאנחנו הולכים לבנות שרת באינטרנט. איך נעשה שאפשר יהיה לגשת לשרת שלנו מכל מחשב בעולם?
בעיקרון, אנחנו צריכים לשים את התוכנה שלנו על מחשב שתמיד דלוק ושיש לו כתובת חיצונית (ולא רק ברשת הפנימית של ביה"ס). אנחנו יכולים לשלם על מחשב שלם, אבל מצד שני, אנחנו יכולים גם להיעזר בשירות חינמי.
אז כאן משתלב לנו שירות נחמד מאוד שנקרא הרוקו (heroku). היוצרים של הרוקו ייתנו לנו מקום במחשבים שלהם לשים בו את הקוד שלנו ככה שיהיה זמין לכולם באינטרנט. אם נרצה להרים אתר גדול שיקבל מיליוני משתמשים, נצטרך לשלם להרוקו. אבל כל עוד שהשרת שלנו קטן יחסית, הם ייתנו לנו לעבוד בחינם.
  1. לפני הכל, (אם עדיין לא עשיתם את זה), בואו נשמור את הקוד שלנו בגיט - נצטרך את זה בהמשך. תריצו את השורות הבאות ב-bash בתיקיה שבה שמור הקובץ של השרת: 
    git init
git add helloflask.py
git commit -m "first commit"
  2. בואו נתחיל, תפתחו חשבון בהרוקו כאן ואז תתקינו את הכלים שלו לווינדוז מהעמוד הזה
    אגב, החלק בשיעור מכאן והלאה הוא קצת מורכב מהרגיל, אז אל תהססו לבקש עזרה ממני או מחברים.
  3. תריצו את השורה הבאה ב-bash ותזינו את הפרטים שלכם בהרוקו כדי לקשר בין המחשב שלכם לחשבון שיצרתם: 
    heroku login
  4. זוכרים את השורה app.run(debug=True)? זאת שורה שאנחנו רוצים שתהיה רק אם אנחנו מריצים את השרת בעצמנו, אז בואו נשנה אותה ונוסיף בדיקה, כך שהיא לא צרוץ בהרוקו אלא הוא יוכל להריץ את השרת שלנו בדרך שלו.
    למטה, זה הקובץ עם התיקון. שימו לב שאם אתם כבר עשיתם שינויים מגניבים משלכם, אתם רק צריכים לשנות את הסוף של הקובץ. 
    import flask

app = flask.Flask(__name__)

@app.route('/')
def hello_world():
    return 'Hello World!'

@app.route('/hello/<name>')
def hello(name):
    return "Hello there <b>%s</b>" % name

@app.route('/bye/<name>')
def bye(name):
    return "Goodbye <i>%s</i>" % name

@app.errorhandler(404)
def page_not_found(error):
    return "404: You're funny, there's no such page", 404

if __name__ == "__main__":
    app.run(debug=True)
  5. תיצרו קובץ בשם Procfile באותה תיקיה ושימו בו את השורה הבאה: 
    web: gunicorn helloflask:app
    הקובץ הזה אומר להרוקו מה הוא צריך להריץ. במקרה הזה, אנחנו מבקשים ממנו לפתוח את הקובץ helloflask (לא צריך לכתוב לו את הסיומת .py) ובתוכו לקחת את האפליקציה app.
  6. כמו-כן, תיצרו קובץ בשם requirements.txt באותה תיקיה ושימו בו את השורה הבאה: 
    Flask==0.9
Jinja2==2.6
Werkzeug==0.8.3
gunicorn==0.17.2
    הקובץ הזה מספר להרוקו איזה ספריות נוספות (ובאיזה גרסאות) הוא צריך בשביל להריץ את השרת.
  7. בואו נוסיף לגיט את כל השינויים שעשינו. תכתבו ב-bash את הפקודות הבאות: 
    git add helloflask.py Procfile requirements.txt
git commit -m "prepared for heroku"
  8. עכשיו בשביל להפעיל את השרת שלנו באינטרנט בפעם הראשונה, תריצו ב-bash את הפקודות הבאות: 
    heroku create
git push heroku master -u
heroku ps:scale web=1
heroku open
    בכך סיפרנו להרוקו שאנחנו רוצים להרים שרת חדש, דחפנו לו את הקוד שכתבנו בעזרת גיט ואפילו ביקשנו שיפתח לנו את העמוד הראשי של האתר שלנו בדפדפן.
    הכתובת הזאת שקיבלתם בדפדפן תעבוד עכשיו מכל מכשיר שמחובר לאינטרנט. תנסו להיכנס אליה ממחשב אחר או אפילו מסמארטפון.
  9. בשיעור קודם השתמשנו בגיט בשביל לשמור את הקוד שלנו בצורה מאורגנת במחשב שלנו ובגיטהאב. כמו שסיפרתי אז, גיט שימושי מאוד בשביל לנהל את הקוד ולשתף אותו עם חברים, אבל עכשיו ראינו שכמו שדוחפים את הקוד שלנו לגיטהאב אפשר באותה הקלות גם לדחוף את הקוד לשרת שלכם. גיט שימושי להמון דברים ואגב אני נעזר בו כדי להעלות לאינטרנט את השיעורים האלו שאתם קוראים.
  10. בואו נעשה שינוי בקוד שלנו ונראה עד כמה קל להעלות את האתר המעודכן לשרת שלנו בהרוקו.
    בואו נכתוב פונקציה חדשה, למשל תוסיפו לקובץ helloflask.py את: 
    
@app.route('/how_is_git')
def how_is_git():
    return '<h1>Git is awesome!</h1>'
  11. ועכשיו תוסיפו את השינוי לגיט ותדחפו אותו לשרת בהרוקו בעזרת הפקודות הבאות ב-bash: 
    git add helloflask.py
git commit -m "added a page about how awesome git is"
git push
    זהו. ככה מרימים שרת באינטרנט. כמובן שבהמשך אנחנו נלמד עוד דברים, הן לגבי פייתון והן לגבי עבודה עם האינטרנט, אבל מה שלמדנו היום יכול להספיק לכם כדי להרים אתר לא רע בכלל. אשמח שתפתיעו אותי באתר שהכנתם בעצמכם.
